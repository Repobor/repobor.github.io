目录和文件结构 #

右上角的导航菜单是自动生成的 #

会在项目启动时，自动生成。如果在项目启动之后，新建了目录，那必须重启，才会重新生成。

也支持监听文件夹创建，从而自动生成（适合写大量文章时使用脚本：watchChangeRestartVitepress）

左侧的文章分类也是自动生成的 #

会在项目启动时，自动生成。如果在项目启动之后，新建了目录，那必须重启，才会重新生成。

也支持监听文件夹创建，从而自动生成（适合写大量文章时使用脚本：watchChangeRestartVitepress）

已集成本地搜索 #

本地启动，搜索索引只会在每次启动项目时创建。打包时也会重新建立搜索索引。

已集成图片点击查看功能 #

并支持鼠标滚动放大/缩小图片

目录与文件 #

目录可以是中文名

index.md：该文件用于定制首页
docs
|--articles : 文档根目录：该目录下的所有子目录，都会成为右上角的导航菜单，目录名前加`01-`这种格式的前缀，可以控制目录在导航菜单上的顺序
|----|----nav: 右上角导航目录：该目录下的所有子目录，都会展现在左侧，目录名前加`01-`这种格式的前缀，可以控制目录在左侧的展示顺序
|----|----|----index.md：该文件对于每个nav目录都是必须的，里面可以写内容，但必须存在
|----|----|----classify: 文章分类目录：该目录下放具体的文章，文件名会作为文章名，展现在左侧
|----|----|----|---demo：所有叫demo的目录，都用于存放代码可供<demo src=""></demo>标签引用
|----|----|----|---asserts: 所有叫asserts的目录，都用于存放静态资源。如：图片，然后可以在文章中使用相对路径的方式引用
|--public ：静态资源目录：用于存放静态资源（如：图片）。所有该目录下的静态资源，都可以通过绝对路径的方式引入。
|--.vitepress ：vitepress配置相关目录
